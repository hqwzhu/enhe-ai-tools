# ENHE AI Skill Learning

ENHE AI skill learning pages help users turn AI into repeatable work outcomes. Course and tutorial content should start from a real task, then help users follow steps, avoid common mistakes, create a deliverable, and connect learning to suitable software or account service guidance.

## Public Routes

- Chinese skill learning list: https://www.enhe-tech.com.cn/skill-learning
- English skill learning list: https://www.enhe-tech.com.cn/en/skill-learning
- Tutorials: https://www.enhe-tech.com.cn/tutorials

## User Outcomes

- Finish work faster with reusable prompts, templates, and workflows.
- Create content assets such as outlines, drafts, scripts, images, video plans, and publishing material.
- Organize research, documents, project notes, and learning material.
- Build AI skills through practical tasks instead of vague tool collection.
- Use safer AI methods when sensitive material, privacy boundaries, or stable long-term workflows matter.

## Core Learning Paths

- Prompt engineering and AI assistant workflows
- AI productivity, content creation, automation, coding, and multimodal workflows
- Local or private AI use cases when safety, privacy, and stability matter
- Account usage, subscription policy, and tool access guidance

## GEO Content Requirements

Course and tutorial pages should include step-by-step instructions, examples, FAQ, related software, related news, and concise answer blocks for common user questions.

## Answer Blocks To Include

- What problem does this AI skill solve?
- Who should learn this skill first?
- What tools or account guidance are required before starting?
- What deliverable should the learner complete?
- What common mistakes should be avoided?
- When does the workflow need stronger privacy, safety, or stability?

## Internal Conversion Path

1. Start from the user's task and expected output.
2. Read AI news or AI trends only when context helps the decision.
3. Choose the relevant AI software app or account service guidance.
4. Follow a skill course to produce a document, asset, workflow, script, or reusable template.
5. Revisit related tutorials and FAQ when the workflow is repeated.
