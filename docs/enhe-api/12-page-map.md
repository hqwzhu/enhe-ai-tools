# ENHE API Gateway Page Map

Date: 2026-07-05

Status: Phase 1 planning draft

## 页面架构原则

- 公开页面、用户控制台、管理后台均使用 ENHE 自有品牌、独立 UI、独立文案和独立代码实现。
- 公开页面和控制台放在现有 ENHE 网站中，依据阶段 0 审计的复用结论。
- 运行时 `/v1/*` 不在页面地图中，它属于独立 Gateway 服务。
- 页面验收必须覆盖空状态、错误状态和权限要求，避免后续实现只完成 happy path。

## 公开页面

### `/ai-api`

| 字段 | 内容 |
| --- | --- |
| 页面目标 | 介绍 ENHE API Gateway 的定位、适用人群、接口能力、工具接入和进入控制台路径。 |
| 主要用户 | 未登录新用户、已有 ENHE 用户、AI 开发者、AI 编程工具用户。 |
| 入口来源 | 站点导航、首页模块、SEO、文档链接、推荐链接。 |
| 核心模块 | 产品价值、支持接口、适用工具、快速开始、套餐入口、合规提示、登录/开通按钮。 |
| 数据来源 | 静态文案、公开模型摘要、公开套餐摘要、站点导航组件。 |
| 用户操作 | 登录/注册、进入控制台、查看价格、查看文档、复制公开 Base URL。 |
| 空状态 | 公开模型或套餐未配置时展示“即将开放”与等待名单入口。 |
| 错误状态 | 公开配置加载失败时保留核心介绍和登录入口。 |
| 权限要求 | 公开可访问；进入控制台需登录。 |
| 验收标准 | 页面不使用第三方品牌/文案；CTA 能正确跳转；未登录用户登录后回到 API 控制台。 |

| 字段 | 内容 |
