# ENHE API Gateway Legal And Compliance Extension

Date: 2026-07-05

Status: Phase 2 legal/compliance draft

## 1. 目标

ENHE API Gateway 上线前，现有法律条款需要覆盖 API 服务、模型输出、请求日志、第三方上游供应商、额度退款、内容安全、滥用处理和推荐奖励规则。

本文是法务和产品实现前的条款设计清单，不构成正式法律文本。

## 2. 用户协议新增 API 服务条款

用户协议需要新增：

- ENHE API Gateway 是面向开发者的 API 服务模块。
- 用户负责保管 ENHE API Key。
- 用户不得共享、转售、泄露、出租 API Key。
- 用户通过 API 输入和输出的内容由用户负责。
- ENHE 可因安全、滥用、违法、供应商故障、维护或风控暂停 API 服务。
- 模型和上游 provider 可调整、下线或限流。
- ENHE 可对 API 使用设置速率限制、额度限制、模型权限和风控规则。
- 用户不得绕过计费、限流或安全机制。

## 3. 隐私政策扩展

隐私政策需要说明：

### API 请求日志

MVP 保存请求元数据：

- request id。
- 用户 ID。
- API Key 前缀或内部 ID。
- 请求路径。
- 模型名。
- token 数。
- 状态码。
- 错误码。
- 消耗金额。
- 延迟。
- IP hash。
- User Agent hash。
- 创建时间。

### 是否保存请求正文

MVP 默认不保存：

- prompt。
- messages。
- completion。
- 响应正文。

如 Phase 2 增加请求内容保存：

- 必须由用户显式开启。
- 必须说明用途，例如排障或质量分析。
- 必须说明保留周期和删除方式。
- 管理员查看需要审计。

### IP/User Agent 脱敏

- 默认保存 hash。
- 用于安全审计、限流、异常检测和推荐反作弊。
- 不在用户控制台展示原始 IP 或完整 User Agent。

### 第三方上游模型供应商处理数据

条款需要说明：

- API 请求可能被转发到第三方模型供应商处理。
- 第三方供应商可能按其条款处理输入、输出和元数据。
- ENHE 会尽量选择符合服务要求的供应商，但用户需理解上游依赖。
- 对供应商保留数据、训练使用、区域传输等事项需按供应商条款逐项确认。

## 4. 退款规则扩展

需要覆盖：

| 额度类型 | 退款原则 |
| --- | --- |
| 套餐余额 | 按套餐有效期、赠送属性、已消耗比例和服务条款处理 |
| 充值余额 | 未消耗部分可按支付和退款规则处理 |
| 推荐余额 | 通常不可提现或退现金，可设置有效期 |
| 已消耗额度 | 原则上不退款，系统错误或服务异常另行处理 |
| 服务不可用 | 按影响时长、范围和用户损失提供补偿或延期策略 |

退款条款需明确：

- 已经调用上游并产生可确认 token 的请求如何处理。
- 流式中断、上游失败和待审计请求如何处理。
- 用户违规、密钥泄露或滥用导致的消耗是否退款。
- 账号冻结后的未消耗余额如何查看和处理。

## 5. 免责声明扩展

需要说明：

- 模型输出不保证准确、完整、可靠、适合特定用途。
- 用户应自行验证输出结果。
- 用户对输入内容、输出使用、二次传播和业务决策负责。
- ENHE 不承诺任一模型永久可用。
- 上游供应商故障、网络波动、维护、政策变化可能影响服务。
- 禁止违法、侵权、欺诈、攻击、骚扰、色情、暴力、绕过安全等滥用。

## 6. 上游供应商条款检查清单

每接入一个 provider 前必须确认：

| 检查项 | 需要结论 |
| --- | --- |
| 是否允许代理 | 是否允许 ENHE 代表用户转发 API 请求 |
| 是否允许转售 | 是否允许商业化转售或额度包装 |
| 是否允许 API 中转 | 是否允许以兼容接口形式中转 |
| 是否允许隐藏供应商 | 是否必须向最终用户披露 provider |
| 是否要求告知最终用户 | 是否必须在条款或页面中展示特定说明 |
| 数据保留 | provider 是否保存输入/输出/元数据 |
| 数据训练 | provider 是否可能用于训练或改进 |
| 地区限制 | 是否存在国家、地区或行业限制 |
| 禁止用途 | 是否需要同步 provider 禁止用途 |
| SLA/稳定性 | 是否可承诺给 ENHE 用户 |

未确认允许代理、转售或 API 中转的 provider 不应进入公开收费模型。

## 7. 内容安全和滥用禁止条款

用户不得使用 ENHE API：

- 生成或传播违法内容。
- 侵犯他人隐私、知识产权或商业秘密。
- 进行网络攻击、钓鱼、恶意软件、凭据窃取。
- 绕过平台安全机制或模型安全策略。
- 大规模垃圾信息、刷量、欺诈或操纵。
- 未经授权处理敏感个人信息。
- 将 API Key 转卖、共享或公开发布。
- 使用自动化方式薅赠送额度或推荐奖励。

ENHE 可采取：

- 限流。
- 暂停模型。
- 撤销 Key。
- 冻结 API 能力。
- 拒绝退款。
- 终止服务。
- 保留必要证据并配合监管要求。

## 8. 数据删除与用户注销策略

建议策略：

- 用户可请求注销 ENHE 账号，按现有账号注销流程处理。
- 可删除或匿名化开发者资料中的非必要个人信息。
- API Key hash 可撤销并保留安全审计期。
- 支付、账务、审计日志按法律和财务要求保留。
- usage log 元数据按保留周期删除或匿名化。
- 默认不保存请求正文，因此无正文删除压力；若 Phase 2 保存正文，必须提供删除策略。

## 9. 管理员冻结账号的条款依据

条款需要允许 ENHE 在以下场景冻结 API 能力：

- API Key 泄露或疑似泄露。
- 异常高消耗或疑似自动化滥用。
- 欠费、退款争议或支付风险。
- 违反内容安全或禁止用途。
- 违反上游供应商条款。
- 试图绕过限流、计费或风控。
- 监管、法律或安全要求。

冻结后用户仍应能登录查看必要账单、余额、日志和申诉入口，除非法律或安全要求另有限制。

## 10. 推荐奖励规则与反作弊说明

推荐规则需要说明：

- 邀请链接产生推荐关系不等于立即发放奖励。
- 好友完成验证和首次有效 API 调用后才发放基础奖励。
- 推荐余额可能有有效期、模型范围或使用限制。
- 推荐余额通常不可提现。
- 自邀、多账号、虚假注册、批量注册、共享设备/IP 风险等可能导致奖励冻结或取消。
- ENHE 可对异常推荐进行人工审核。
- 推荐奖励调整应进入额度流水。

## 11. 上线前法律验收

- 用户协议已新增 API 服务条款。
- 隐私政策已覆盖 API 请求日志、正文保存策略、IP/User Agent 脱敏和第三方供应商处理。
- 退款规则已覆盖套餐、充值、推荐、已消耗额度和服务不可用。
- 免责声明已覆盖模型输出准确性和用户责任。
- Provider 条款检查已完成并记录。
- 禁止用途、冻结依据、推荐反作弊规则已公开或在相关页面可达。
