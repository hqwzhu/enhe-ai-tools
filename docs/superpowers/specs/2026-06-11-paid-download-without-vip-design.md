# 取消 VIP 并改为单软件付费下载设计

## 背景

当前网站同时存在两套商业逻辑：

- VIP 套餐：前台会员价格页创建 VIP 订单，后台会员套餐管理，审核通过后生成会员权益。
- 单软件付费下载：工具可设置为付费下载，用户付款审核通过后生成 `ToolPurchase` 授权。

新的产品方向是取消 VIP 售卖和 VIP 权限概念，改为按具体软件付费。用户只有在购买某个收费软件并通过付款审核后，才能在该工具详情页查看下载链接内容。

## 目标

上线后不再向用户展示或引导开通 VIP。网站的付费主路径改为：

1. 管理员新增或编辑本地应用时设置“收费软件”和“收费金额”。
2. 用户进入工具详情页点击“下载软件”。
3. 如果软件免费，用户可直接查看或打开下载链接。
4. 如果软件收费且用户尚未购买，页面提示需要支付费用。
5. 用户确认支付后创建软件购买订单，并跳转到现有收款码支付页。
6. 用户上传付款凭证。
7. 后台审核通过后，系统生成该用户对该软件的下载授权。
8. 用户返回工具详情页，可查看下载链接内容。

## 非目标

- 本次不接入微信 Native 支付自动回调。
- 本次不物理删除 `VipPlan`、`Membership`、`VipAdjustmentLog` 等历史表。
- 本次不迁移或清空旧 VIP 订单、旧会员记录。
- 本次不重做支付页视觉，只沿用现有收款码和凭证审核流程。

## 产品边界

### 前台

- 顶部导航删除“会员价格”。
- 移动端菜单删除“会员价格”。
- 工具详情页不再展示 VIP 标签、开通 VIP 按钮、VIP 购买文案。
- 工具卡片不再展示 VIP 角标。
- 用户中心不再展示“我的会员”卡片，不再引导查看会员套餐。
- 用户中心保留订单、已购软件、下载记录、云端工具使用记录、评论、账号设置。
- `/pricing` 不作为付费入口。为了兼容旧链接，页面改为提示“会员套餐已下线，请在具体工具页购买收费软件”，并提供前往本地应用的入口。

### 后台

- 后台导航删除“会员套餐”。
- 后台工具编辑页删除“需要 VIP”和“下载链接仅 VIP 可见”两个控制项。
- 后台工具编辑页保留“下载单独付费”和“下载价格”。
- 本地应用列表的权限列改为显示“免费软件”或“收费 ¥xx”，不再显示 VIP。
- 后台看板删除“有效 VIP”和“7 天内到期 VIP”指标。
- 管理员消息中心删除 VIP 到期提醒。
- 用户管理页面不再突出会员状态；历史会员记录可以保留在数据库但不作为运营入口展示。

## 权限与数据模型

### 保留

- `Order.orderType = software_download`
- `Tool.isDownloadPaid`
- `Tool.downloadPrice`
- `ToolPurchase`
- `PaymentProof`
- `OrderRefundRecord`
- `DownloadLog`

### 兼容保留

- `Order.orderType = vip`
- `VipPlan`
- `Membership`
- `VipAdjustmentLog`

这些只用于历史数据兼容，不再作为新订单入口。

### 权限规则

- 免费软件：
  - 如果配置了下载链接内容，登录用户可查看下载链接区域。
  - 如果下载链接是可打开地址，允许走现有下载接口并记录下载日志。
- 收费软件：
  - 未购买：不展示下载链接内容，只展示付费提示和购买按钮。
  - 已购买：展示下载链接内容；如果链接可打开，允许下载并记录日志。
- 云端工具：
  - 不再用 VIP 作为访问门槛；发布状态正常即可登录后使用。

## 订单与审核流程

### 创建订单

用户在收费软件详情页确认购买后调用现有 `createSoftwareDownloadOrderAction`：

- `orderType = software_download`
- `toolId = 当前工具`
- `amount = tool.downloadPrice`
- `paymentMethod = 用户选择的支付宝或微信`
- `orderStatus = pending_payment`

### 支付页

继续使用 `/orders/[id]/pay`：

- 展示订单号、工具名称、金额、收款码。
- 用户上传付款截图。
- 上传后订单变为 `pending_review`。

### 后台审核

继续使用现有支付审核入口：

- 审核通过：创建或更新 `ToolPurchase`，订单变为 `activated`。
- 审核拒绝：订单变为 `rejected`。
- 审核文案从“开通权益/VIP”调整为“开通软件下载权限”。

### 退款

退款完成后撤销该软件的 `ToolPurchase` 授权。

## 文案调整

所有直接面向用户的新业务文案统一为：

- “收费软件”
- “购买下载”
- “支付后查看下载链接”
- “付款审核通过后即可查看下载链接内容”
- “已购软件”

避免再出现：

- “VIP”
- “会员套餐”
- “开通 VIP”
- “会员价格”
- “下载链接仅 VIP 可见”

历史订单详情中如果遇到旧 VIP 订单，可以显示为“历史会员订单”，但不提供新的购买入口。

## 测试要求

至少覆盖以下测试：

1. 顶部导航和移动端菜单不包含 `/pricing` 会员价格入口。
2. 后台导航不包含 `/admin/plans` 会员套餐入口。
3. 工具详情页对未购买收费软件不展示下载链接内容。
4. 工具详情页对已购买收费软件展示下载链接内容。
5. 免费软件可展示下载链接内容。
6. 付款审核通过 `software_download` 订单后创建 `ToolPurchase`。
7. 退款完成后撤销 `ToolPurchase`。
8. 关键文案不再出现 VIP 售卖入口。

## 验收标准

- 新用户从首页进入网站，看不到会员价格入口。
- 管理员从后台进入，看不到会员套餐入口。
- 管理员能在本地应用里设置软件是否收费和收费金额。
- 未购买收费软件的用户点击下载时，会被引导创建软件购买订单。
- 付款审核通过后，用户可以在对应工具详情页查看下载链接内容。
- 已购买授权只对当前软件生效，不影响其他收费软件。
- 旧订单和旧数据库记录不导致页面报错。
