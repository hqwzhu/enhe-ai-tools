# AI News Auto Publishing Design

Date: 2026-06-18

## Goal

When ENHE AI news articles are generated by the `enhe-ai-news-seo` workflow, the article can be published into the existing ENHE website news system automatically. After import, the article must appear in the admin AI news module, where admins can review, edit, publish, archive, pin, feature, and update it through the existing UI.

The first version should favor editorial safety: automatically imported articles are created as drafts by default, with an explicit option to import as published when the caller requests it and has the import token.

## Recommended Architecture

Add a protected import endpoint plus a local publishing script:

1. Generated article data is normalized into a structured JSON payload.
2. A local script sends that payload to the website.
3. A server-side import API validates the bearer token and payload.
4. The API creates or updates `NewsArticle`, `NewsExternalSource`, `NewsTag`, and `NewsArticleTag`.
5. The article appears in `/admin/ai-news` immediately.
6. Public pages are revalidated when the imported article is published.

This avoids browser automation against the admin UI. It is more stable, testable, and secure than logging into the admin panel and filling fields automatically.

## Scope

### In Scope

- Add a protected API route:
  - `POST /api/admin/ai-news/import`
- Add an environment variable:
  - `AI_NEWS_IMPORT_TOKEN`
- Add a reusable import service for validating, normalizing, and saving generated news.
- Add a local script for publishing generated article JSON:
  - `scripts/publish-ai-news.ts`
- Store automatic import metadata on `NewsArticle`.
- Create or reuse tags and categories during import.
- Store verified source links in `NewsExternalSource`.
- Write an `AdminAuditLog` entry for each import.
- Revalidate admin and public news routes after import.
- Add tests for authentication, validation, draft import, published import, slug uniqueness, tags, sources, and audit logging.

### Out Of Scope

- Fully automated scheduled news research and generation.
- Browser automation for admin form filling.
- WYSIWYG editing.
- Multi-article batch scheduling UI.
- Email newsletter dispatch after import.
- AI provider integration inside the website server.

The website should accept finished editorial payloads. The research and article generation workflow can continue to happen in Codex or another trusted external worker.

## Data Model Changes

Extend `NewsArticle` with optional import metadata:

- `sourceChannel String? @map("source_channel")`
- `importedAt DateTime? @map("imported_at")`
- `importBatchId String? @map("import_batch_id")`
- `rawImportPayload Json? @map("raw_import_payload")`

Recommended values:

- `sourceChannel`: `ai_auto_import`
- `importBatchId`: a stable caller-provided ID, or a generated timestamp-based ID.
- `rawImportPayload`: sanitized JSON payload for traceability. It must not include secrets.

Indexes:

- `@@index([sourceChannel, importedAt])`
- `@@index([importBatchId])`

These fields make imported records easy to find later without changing the current admin editing flow.

## Publishing Modes

### Default: Draft

`publishMode: "draft"`

Behavior:

- Creates a `draft` article.
- Leaves `publishedAt` null.
- Shows in admin list.
- Does not appear publicly.
- Returns an admin edit URL.

This is the recommended default because generated news may still need human review.

### Explicit Published Import

`publishMode: "published"`

Behavior:

- Creates a `published` article.
- Sets `publishedAt`.
- Revalidates public list and detail routes.
- Returns admin and public URLs.

This mode should be used only when the generated article has already passed source verification and editorial checks.

## Validation Rules

Required fields:

- `article.title`
- `article.summary`
- `article.content`
- `article.externalSources`

Content rules:

- `content` must be Markdown-like plain text, not raw HTML.
- Reject `<script>`, `<style>`, inline event handlers, and full HTML documents.
- Allow headings, paragraphs, lists, quotes, and plain URLs in the existing renderer format.
- Require at least one external source with a valid `https://` URL.
- Reject source URLs that are not HTTP or HTTPS.
- Limit title, summary, SEO description, and keyword lengths to practical CMS sizes.
- Cap `rawImportPayload` size to avoid storing oversized generation logs.

Slug behavior:

- If a slug is supplied, normalize it with the existing slug helper.
- If no slug is supplied, generate from title.
- If the slug already exists, append a short suffix using the existing unique slug logic.

Category behavior:

- If `categorySlug` matches an existing category, use it.
- If `categorySlug` is missing, use or create `AI快讯` with slug `ai-news-flash`.
- If `categoryName` is supplied for a missing `categorySlug`, create the category as active.

Tag behavior:

- Create missing tags with active status.
- Link all normalized tags to the article.

## Admin Visibility

Imported articles must appear in the existing admin list at `/admin/ai-news`.

Recommended UI improvement:

- Add a small badge in the admin list for imported articles:
  - `自动导入`

The badge can be derived from:

- `sourceChannel === "ai_auto_import"`

The existing editor does not need a separate editing path. Admins should edit imported articles through the current form.

## Audit Logging

Each successful import writes one audit log:

- `action`: `news_article.auto_import`
- `targetType`: `news_article`
- `targetId`: imported article ID
- `summary`: `Auto-imported AI news article.`
- `metadata`:
  - `title`
  - `slug`
  - `status`
  - `sourceChannel`
  - `importBatchId`
  - `sourceCount`
  - `tagCount`

Because import requests use a token rather than a logged-in admin session, `adminId` may be null. The action name and metadata must make the automated origin clear.

## Local Publishing Script

Add:

```text
scripts/publish-ai-news.ts
```

Usage:

```bash
npx tsx scripts/publish-ai-news.ts --file output/ai-news/article.json --mode draft
```

Environment variables:

- `AI_NEWS_IMPORT_TOKEN`
- `AI_NEWS_IMPORT_URL`

Example:

```bash
AI_NEWS_IMPORT_URL="https://www.enhe-tech.com.cn/api/admin/ai-news/import"
AI_NEWS_IMPORT_TOKEN="..." 
npx tsx scripts/publish-ai-news.ts --file output/ai-news/article.json --mode draft
```

The script should:

- Read a JSON file.
- Add or override `publishMode` from `--mode`.
- Send a POST request to the import endpoint.
- Print the returned admin URL and public URL.
- Exit non-zero when import fails.

## Integration With ENHE AI News SEO Workflow

The article generator should produce a structured JSON payload for publishing, not only an HTML article.

Recommended output files:

- `output/ai-news/YYYY-MM-DD-ai-news-slug.html`
- `output/ai-news/YYYY-MM-DD-ai-news-slug.json`

The HTML remains useful for CMS preview and archival. The JSON is the source used by the website import API.

The JSON `content` field should be Markdown-like text compatible with the current `renderNewsContentBlocks` helper. The API should not import the full no-CSS HTML as the article body because the public site already has its own page layout and safe renderer.

## Security

- The import endpoint must be server-only and must not accept browser cookies as authorization.
- Requests must include `Authorization: Bearer <token>`.
- Compare tokens using constant-time comparison where practical.
- Return `401` for missing or invalid tokens.
- Do not log the token.
- Do not store secrets in `rawImportPayload`.
- Keep the token outside the repository in environment variables.
- Production should use a long random token.

Optional later hardening:

- IP allowlist for import callers.
- Request timestamp and HMAC signature.
- Rate limiting.
- Separate token rotation table.

These are useful later, but a strong bearer token is enough for the first version because the endpoint only accepts structured editorial content and is not public-facing in the admin UI.

## Revalidation

After import:

- Always revalidate:
  - `/admin/ai-news`
- If status is `published`, revalidate:
  - `/ai-news`
  - `/en/ai-news`
  - `/ai-news/{slug}`
  - `/en/ai-news/{slug}`

Draft imports should not revalidate public detail pages because they are not publicly visible.

## Error Handling

Return structured JSON errors:

```json
{
  "ok": false,
  "error": "VALIDATION_ERROR",
  "message": "article.externalSources must contain at least one valid source URL"
}
```

Error categories:

- `UNAUTHORIZED`
- `VALIDATION_ERROR`
- `IMPORT_FAILED`

The local publishing script should print the error message and exit with code `1`.

## Testing Strategy

Unit tests:

- Token validation accepts the configured token.
- Token validation rejects missing and incorrect tokens.
- Payload validation rejects missing title, summary, content, and sources.
- Payload validation rejects raw HTML documents and script/style content.
- Slug generation creates a unique slug.
- Category upsert uses `AI快讯` fallback.
- Tags are created and linked.
- External sources are created in order.
- Draft imports create non-public articles.
- Published imports set `publishedAt`.
- Audit log metadata includes import source information.

Route/source tests:

- `src/app/api/admin/ai-news/import/route.ts` exists.
- Import route does not call `requireAdmin`.
- Import route checks bearer token.
- Import route uses the shared import service.

Manual verification:

- Import a draft article locally.
- Confirm it appears in `/admin/ai-news`.
- Open `/admin/ai-news/{id}` and edit it.
- Publish it from the admin editor.
- Confirm `/ai-news/{slug}` loads publicly.
- Import a published article and confirm public route appears immediately.
